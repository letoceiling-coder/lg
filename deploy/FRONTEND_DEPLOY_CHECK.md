# Проверка: на сервере новая админка или старая сборка

## Признак старой сборки (до медиа / ручных квартир)

- **Медиа:** текст зоны загрузки *«Перетащите файлы сюда или нажмите „Загрузить“»*, выпадающий список *«Все папки»*, **нет** дерева папок «Загрузки» / «Корзина» и поля «Новая вложенная папка».
- **Квартиры:** кнопка «Ручная квартира» открывает **модальное окно**, а не переход на URL `/admin/listings/manual/new`.

В текущем коде репозитория старых строк **нет** — если вы их видите, nginx отдаёт **старый** каталог `apps/web/dist` или браузер держит старый `index.html`.

## После успешного деплоя

1. В левой колонке админки, **над** кнопкой сворачивания меню, появится строка **«Сборка: …»** (ISO-время сборки Vite). Она меняется при каждой новой сборке.
2. Жёсткое обновление страницы: **Ctrl+Shift+R** (или очистка кэша для сайта).
3. На сервере проверить дату файлов:

```bash
ls -la /var/www/lg/apps/web/dist/index.html
head -c 500 /var/www/lg/apps/web/dist/index.html
```

4. Убедиться, что деплой действительно выполнялся после `git pull`:

```bash
cd /var/www/lg && bash deploy/deploy-from-git.sh
```

Лог скрипта должен содержать шаги `Building frontend...` и `pnpm build:web` без ошибок.

## Альтернативный сценарий (rsync с ПК)

Если фронт выкатывается через `deploy/sync-and-deploy.sh`, на машине перед скриптом должна быть актуальная копия репозитория (`git pull`), иначе на сервер уедет старый код.
